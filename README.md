# higashine-discipline-python ♨️🐍

**「最新の負債」を、琥珀色の秩序へ。**

このリポジトリは、AI時代のエンジニアのための「規律（Discipline）」の実証フィールドです。同一の仕様に対し、AIが吐き出す「カオス」と、規律によって統制された「芸術」の差を可視化します。

---

## 🏛 秘密結社の理念

> **「動くは最低条件。保守性こそが、AIを導く人間の誠実さである。」**

AIにコードを書かせる際、人間側に明確な「法（ルール）」がなければ、生成されるのはメンテナンス不能なスパゲッティコードです。本プロジェクトは、AIを単なる「コード生成機」から「最強のアーキテクト」へ変貌させるための規律を提示します。

---

## 🧭 プロジェクトの構造

本リポジトリは、比較検証のために2つの世界線を用意しています。

| ディレクトリ | 状態 | 内容 |
| --- | --- | --- |
| `caos/` | **無規律 (Raw)** | 指示書のみで生成された、密結合でテスト困難なスパゲッティコード。 |
| `discipline/` | **琥珀色の秩序 (Disciplined)** | 「4つの聖典」を遵守し、クリーンアーキテクチャに基づき洗練されたコード。 |
| `docs/` | **法典 (Archives)** | 要件定義書、およびAIを律するための各ガイドライン。 |

---

## 🗂 エージェント・レビュー

各プルリクエスト（PR）では、秘密結社のエージェントたちが独自の視点で「法執行」を行っています。AIによるAIのためのレビューログは必見です。  
別プロジェクトとして用意しておりますので、公開までお待ちください。

* **🚨 規約警察のルル**: 命名規則、ファイル構成、トランザクションの不備を一切逃さない。
* **☕️ アーキテクトのマリ**: 責務の分離、依存関係、クリーンアーキテクチャとしての美しさを審議する。
